# X 大神方法论 —— 卡脖子垄断选股与动态熔断系统

## 概述

四阶段递进式投资研究系统：从赛道 BOM 拆解锁定垄断环节，到财务数据硬核穿透验证，再到空头视角极限证伪，最终用贝叶斯动态熔断机制监控持仓。每一层都是对前一层的收紧过滤器，目标是只留下真正具备"不可替代性 + 扩产壁垒 + 估值未被透支"的隐形冠军。

## 依赖的外部 Skill

执行本 skill 时，主动加载以下已有 skill 获取数据：

| Skill | 用途 |
|-------|------|
| `a-stock-data` | A股行情、财务数据、研报、资金流、互动易 |
| `neodata-financial-search` | 自然语言金融搜索（产业链数据、国产化率、行业动态） |
| `stock-analyzer` | 个股综合分析（基本面+新闻面+资金面） |
| `sepa-vcp-stock-screener` | SEPA 策略辅助筛选（趋势+量价形态） |

## 四阶段工作流

### 阶段 0：数据校验门禁（⚠️ 必须最先执行）

**目标**：在进入任何分析之前，用实时数据验证所有硬性筛选条件。

**铁律：禁止使用训练数据记忆、估算值、带~号的猜测值作为筛选依据。所有数值必须来自 `neodata-financial-search` 实时查询结果。**

**执行步骤（按顺序，不可跳过）**：

1. **候选公司列表初步形成后，立即查实时总市值和流通市值**
   - 使用 `neodata-financial-search` 查询每只股票的 **总市值 + 流通市值**（当日最新）
   - 将实时市值填入表格，**不允许留空或用~估算**

2. **逐项硬性条件过滤**：
   - ❌ 市值 < 30亿 或 > 150亿 → 直接剔除，不进入后续阶段
   - ❌ ST / *ST → 剔除
   - ❌ 上市 < 2 年的次新股 → 剔除
   - ❌ 最近一年有财务造假/监管处罚记录 → 剔除

3. **输出「阶段0校验表」**（必须展示给用户）：
   ```
   | 代码 | 名称 | 实时总市值(亿) | 流通市值(亿) | 30-150亿? | ST? | 次新? | 通过? |
   ```
   - 数据来源标注：`neodata-financial-search 查询时间: YYYY-MM-DD HH:MM`
   - 未通过项必须标红并说明原因

4. **只有通过阶段0的标的，才允许进入阶段1的分析流程。**

**常见错误警示**：
   - AI概念股涨幅巨大，旧估值可能偏离当前市值数倍甚至数十倍。必须每次查询实时数据。
   - 科创板股票波动剧烈，昨日市值≠今日市值。
   - "专精特新"标签不等于小市值——很多已成长为中大盘股。

### 阶段 1：月度初筛 —— 锁定垄断环节

**目标**：从赛道 BOM 中找出同时满足"国产化率极低 + 成本占比极小 + 扩产极难"的环节，匹配专精特新冠军。

**⚠️ 重要：阶段1输出的所有候选标的，必须已经通过阶段0的实时数据校验。**

**前置准备**：
- 加载 `references/bom_analysis_guide.md` 了解 BOM 拆解和三维打分方法
- 加载 `references/localization_reference.md` 获取各赛道国产化率参考数据

**执行步骤**：

1. **赛道定义**：向用户确认目标赛道（如：人形机器人、半导体设备、碳化硅等）

2. **BOM 逆向拆解**：
   - 使用 `a-stock-data` 和 `neodata-financial-search` 获取该赛道的产业链深度研报
   - 逐级拆解终端产品的 BOM，列出关键零部件/材料
   - 对每个环节标注：国产化率、成本占比、扩产周期

3. **垄断度计算**（参考 `scripts/monthly_screener.py` 中的 `score_monopoly` 函数）：
   - 过滤条件：国产化率 < 20% **且** 成本占比 < 5% **且** 扩产周期 > 18 个月
   - 三维加权：国产化率 40% + 成本占比 30% + 扩产周期 30%，满分为 5

4. **公司匹配**：
   - 用 `neodata-financial-search` 搜索每个高分环节对应的 A 股公司
   - 只保留：专精特新小巨人 / 制造业单项冠军
   - 市值约束：30 亿 - 150 亿（**必须使用 neodata-financial-search 当日实时查询值**）
   - 排除：ST、次新股（上市 < 2 年）

5. **输出**：按垄断得分排序的候选池表格

**输出格式**：

```
| 排名 | 代码 | 名称 | 赛道 | 环节 | 国产化率 | 成本占比 | 扩产周期 | 得分 | 实时市值(亿) | 专精特新 | 单项冠军 |
```

> **注**：市值列必须标注为「实时市值」，附带查询时间和数据来源。禁止使用估算值。

### 阶段 2：季度复审 —— 财务硬核穿透

**目标**：对月度初筛候选池进行财务验证，找出"主营驱动真实增长 + 正在秘密扩产 + 资金投向卡脖子"的标的，排除研报过热的。

**前置准备**：
- 加载 `references/financial_indicators.md` 了解财务指标体系和判定标准

**执行步骤**：

1. **财务数据采集**（对候选池每只股票）：
   - 用 `a-stock-data` 获取最近 4 个季度的：营收、毛利率、扣非净利润、CapEx、在建工程、固定资产
   - 计算：毛利率逐季趋势、CapEx 同比增速、在建工程/固定资产比率
   - 使用 `scripts/quarterly_review.py` 中的 `QuarterlyReview` 类辅助计算

2. **三项硬核指标验证**：

   **A. 毛利率连续扩张**：
   - 近 4 季度中至少 3 季度毛利率环比提升
   - 最新季度毛利率较最早季度提升 > 2 个百分点
   - 扣非毛利率与综合毛利率偏离 < 5%

   **B. CapEx / 在建工程秘密爬坡**：
   - 最新季度 CapEx 同比增速 > 30%
   - 在建工程连续 2 季度增加
   - 在建工程/固定资产 > 15%

   **C. 定增投向卡脖子环节**（如有）：
   - 搜索近 12 个月定增预案/实施公告
   - 审查募集资金用途是否指向卡脖子环节
   - 关注是否有产业链客户/国家大基金参与

3. **研报过热度排除**：
   - 用 `a-stock-data` 统计近 3 个月研报数量
   - 研报 > 15 篇 → 直接剔除（信息差已被充分消化）

4. **输出**：通过复审的"硬核候选池"

**输出格式**：

```
| 排名 | 代码 | 名称 | 毛利趋势 | 毛利提升 | 最新毛利 | CapEx增速 | 在建/固资 | 定增投向 | 研报数 | 结论 |
```

### 阶段 3：买前终审 —— AI 红队空头证伪

**目标**：扮演极度苛刻的空头分析师，从三个维度对标的进行路径证伪。任一维度亮红灯 → 直接回避。

**前置准备**：
- 加载 `references/short_seller_framework.md` 了解空头证伪框架和判定标准

**执行步骤**：

对季度复审通过的每只标的，执行三维证伪：

**维度一：大客户路径证伪**

需回答以下问题并给出红/黄/绿灯：
1. 前五大客户收入占比是否 > 60%？最大单一客户是否 > 40%？
2. 大客户是否有自研计划？（查大客户专利、招聘、年报中"自主可控"关键词）
3. 大客户是否在引入二供？（查供应链新闻、竞品公告）
4. 认证切换周期有多长？（> 18 个月为安全）

数据来源：
- 大客户年报/ESG 报告（WebSearch 搜索）
- 招聘网站（搜索大客户相关岗位）
- 国家知识产权局专利检索
- `neodata-financial-search` 搜索供应链动态
- `a-stock-data` 的互动易模块查询投资者问答

**维度二：技术路径证伪**

需回答以下问题并给出红/黄/绿灯：
1. 当前技术处于第几代？下一代是否已在实验室阶段？
2. 是否存在替代技术方案？成本/性能对比如何？
3. 替代技术的产业化时间表？（< 18 个月则危险）
4. 核心专利保护期和竞品绕专利可能性？

数据来源：
- 学术论文（arXiv、CNKI、Google Scholar）
- 专利数据库（Google Patents、国家知识产权局）
- WebSearch 搜索行业技术路线图
- 海外竞品公司技术公告

**维度三：竞争格局证伪**

需回答以下问题并给出红/黄/绿灯：
1. 同行新建同样产能需要多久？（< 18 个月则壁垒低）
2. 产能是否受出口管制/特殊资质限制？
3. 如果价格降 30%，公司能否盈利？（毛利率安全线）
4. 新进入者获得大客户认证需要多长时间？

数据来源：
- 竞品公司年报/招股书
- `neodata-financial-search` 搜索产能数据
- 设备供应商出货数据（反推产能扩张）

**综合裁决**：
- 任一维度红灯 → 直接回避
- 两个维度黄灯 → 回避
- 一个黄灯 + 两个绿灯 → 可小仓位（< 5% 总仓位）
- 全部绿灯 → 正常建仓（< 10% 总仓位）

**输出格式**：

```
| 证伪维度 | 风险等级 | 核心风险点 | 概率评估 | 应对策略 |
|----------|----------|------------|----------|----------|
```

### 阶段 4：持股监控 —— 贝叶斯动态熔断

**目标**：只基于"逻辑"而非"价格"进行止盈止损。设定硬核里程碑，到期未确认则逻辑失效。

**前置准备**：
- 加载 `references/circuit_breaker_guide.md` 了解贝叶斯熔断框架
- 使用 `assets/circuit_breaker_template.md` 作为输出模板

**执行步骤**：

1. **设定里程碑**（每只持仓标的 3 个）：
   - 类型：技术突破 / 产能落地 / 订单确认 / 产品打样 / 国产替代
   - 必须：可验证（互动易/公告/新闻）+ 有明确时间节点 + 与投资逻辑强相关
   - 时限：未来 2 个季度内

2. **设定熔断规则**：
   - 逻辑可信度 < 30% → 无条件卖出
   - 逻辑可信度 < 50% → 减仓一半
   - 任一里程碑逾期未确认 → 逻辑失效，卖出

3. **输出贝叶斯动态熔断监控表**（严格按照 `assets/circuit_breaker_template.md` 格式）：

```
以 "事件：最晚时限：熔断阈值" 结构输出表格
```

参考 `scripts/circuit_breaker.py` 中的 `format_monitor_table()` 方法。

4. **定期更新**：
   - 每周检查互动易问答更新
   - 每月跟踪行业新闻/竞品动态
   - 每季财报后重新评分
   - 里程碑截止日前一周重点提醒

## 使用 `scripts/` 脚本

三个脚本是 Python 辅助工具，提供打分模型和格式化逻辑。执行前确保 Python 环境可用：

```bash
python scripts/monthly_screener.py     # BOM拆解+垄断打分
python scripts/quarterly_review.py      # 财务指标计算
python scripts/circuit_breaker.py       # 贝叶斯熔断监控表
```

注意：脚本仅处理计算和格式化逻辑，实际数据仍需通过 `a-stock-data`、`neodata-financial-search` 等 skill 实时获取。

## 使用 `references/` 参考文档

当需要深入了解某一阶段的方法论时，按需加载以下文档：
- `references/bom_analysis_guide.md` — BOM 拆解方法和垄断打分模型
- `references/localization_reference.md` — 各赛道国产化率参考数据（需交叉验证）
- `references/financial_indicators.md` — 财务穿透指标体系
- `references/short_seller_framework.md` — AI 红队空头证伪框架
- `references/circuit_breaker_guide.md` — 贝叶斯熔断机制原理

## 使用 `assets/` 资产

- `assets/circuit_breaker_template.md` — 贝叶斯动态熔断监控表模板。当用户需要输出熔断表时，复制此模板并填入实际数据。

## 快速开始示例

典型对话流程：

1. 用户："帮我筛一下人形机器人赛道的卡脖子垄断环节"
   → 执行阶段 1（月度初筛）

2. 用户："对柯力传感、绿的谐波做季度财务穿透"
   → 执行阶段 2（季度复审）

3. 用户："对绿的谐波做空头证伪，看看有没有风险"
   → 执行阶段 3（买前终审）

4. 用户："帮柯力传感设 3 个硬核里程碑，做贝叶斯熔断监控表"
   → 执行阶段 4（持股监控）
