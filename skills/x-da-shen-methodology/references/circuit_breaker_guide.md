# 贝叶斯动态熔断机制

## 设计理念

传统止损基于价格，贝叶斯熔断基于"逻辑"。不是股价跌了就卖，而是"当初买入的逻辑被事实推翻就卖"。

## 贝叶斯框架

### 先验概率（买入时）
基于月度初筛 + 季度复审 + 买前终审的分析，形成初始逻辑可信度。

### 后验更新（持股期间）
每当出现新信息（财报、公告、互动易、行业新闻），更新逻辑可信度：

```
更新公式（简化）：
新可信度 = 旧可信度 × 似然比

其中：
- 利好信息 → 似然比 > 1（提升可信度）
- 中性信息 → 似然比 ≈ 1（不变）
- 利空信息 → 似然比 < 1（降低可信度）
```

### 熔断阈值
- 逻辑可信度降至 < 30% → 无条件卖出
- 逻辑可信度降至 < 50% → 减仓至一半
- 里程碑到期未确认 → 逻辑失效，卖出

## 硬核里程碑设定

### 里程碑设计原则
1. **可验证**：必须能在公开渠道（互动易、公告、新闻）确认
2. **有明确时间节点**：不是"未来会做"，而是"X月前完成"
3. **与投资逻辑强相关**：里程碑直接验证卡脖子环节的突破
4. **不可逆**：一旦达成，竞争壁垒确认；一旦逾期，逻辑证伪

### 里程碑类型

| 类型 | 示例 | 验证渠道 |
|------|------|----------|
| 技术突破 | 某型号产品通过大客户认证 | 互动易、公告、客户供应链名单 |
| 产能落地 | 新产线投产/试产 | 公告、当地政府新闻、设备供应商出货 |
| 订单确认 | 获得大客户批量订单 | 公告（达到披露标准）、互动易 |
| 产品打样 | 新产品送样并获认可 | 互动易、行业展会 |
| 国产替代 | 进口替代项目进入量产阶段 | 公告、大客户供应链审计 |

## 监控频率

- **每周**：检查互动易是否有针对该公司的投资者问答更新
- **每月**：跟踪行业新闻、竞品动态
- **每季**：财报发布后重新评分，更新逻辑可信度
- **里程碑截止日前后**：重点监控，提前 1 周提醒

## 熔断执行原则

- 熔断触发后，不犹豫、不幻想、不补仓
- 市场情绪（恐慌/贪婪）不影响熔断执行
- 记录每次熔断的原因，供后续策略迭代
